# CLAUDE.md ‚Äî aurum-mcp

A read-only MCP server that surfaces the **Aurum design system** (components, tokens, icons, Figma-node mappings, Code Connect, changelog) to LLM tooling. It is a *thin renderer* over a single bundled JSON manifest (`data/manifest.json`) generated upstream. Full detail is in `README.md` (`## Versioning`, `## How content gets here`, `## CI workflows`) and `architecture.md` ‚Äî this file is the short list of invariants that are easy to get wrong.

## What this repo is (and isn't)
- **It serves a manifest, it does not author content.** `data/manifest.json` is the **MERGE of two generated manifests** ‚Äî the live gallery manifests of `aurum-android` and `aurum-ios` ‚Äî produced deterministically by `scripts/fetch-manifest.mjs` (fetch both ‚Üí merge ‚Üí same inputs = byte-identical output). **Never hand-edit `data/manifest.json`** ‚Äî run `pnpm manifest:fetch`. Fail-closed: if either gallery is unreachable the fetch exits 1 (no silent partial catalogs). The schema contract (merged fields included) lives in `aurum-android/tooling/manifest/schema.json`; `src/types.ts` is its TypeScript projection ‚Äî keep the two in sync.
- **The manifest is bundled into the server.** `src/manifest.ts` reads the manifest baked into the published package, so the **served catalog only changes when a new version is published** (`release.yml`) ‚Äî committing a fresh manifest to `main` is necessary but not sufficient.

## Versioning (the part people get wrong)
- **aurum-mcp has its OWN SemVer, independent of any source library.** It is an **aggregator** ‚Äî today `aurum-android`, later `aurum-ios`, each on its own cadence. **Never pin the MCP version to a source library version**, and never `npm version` it to match Aurum. The per-tool footer + `aurum_get_aurum_version` already print the source version(s).
- **Multi-platform shape (ACTIVE since 2026-07-22):** the bundled manifest IS merged ‚Äî `aurum.sources` (`{ android: {version,sha}, ios: {version} }`) is authoritative, `aurum.version` mirrors android (primary) for back-compat, overlapping components carry per-platform detail in `component.sources`, codeConnect entries carry `platform`, and per-library changelogs live in `changelogs`. Tools take `platform` args (`get_component`, `get_changelog`, `get_code_connect_snippet`, `list_components`); the footer prints both source versions. Never attribute an android version to ios content. **Session platform lens (src/platform.ts):** those tools DEFAULT their `platform` to the session's repo ‚Äî `AURUM_MCP_PLATFORM` env var wins, else cwd fingerprints (gradlew‚Üíandroid, Package.swift/xcodeproj‚Üíios), else no lens. Explicit args always win; get_component degrades lens‚Üí`all` when the component is missing on the lens platform. The footer appends `¬∑ lens: <p>` when active.

## Drift is the failure mode ‚Äî keep it loud
- The bundled manifest **silently fell ~1 month / 28 icons behind** the gallery once (served 0.1.5, repo 0.2.0, gallery 0.3.5) because **every sync link failed green**: the notify dispatch 403'd, the cron's auto-merge PR never merged (`GITHUB_TOKEN` PRs don't trigger workflows), and `drift-check` only ran on PRs so it never saw `main` fall behind. Result: the MCP told a consumer "no star icon exists" when `AurumIcons.Content.Star` had shipped.
- **Guardrails now:** `sync-manifest.yml` **direct-pushes** the merged manifest to `main` (no PR); `drift-check.yml` runs on a **daily schedule**, **re-runs the merge** against both live galleries (bytes can't be compared to one source anymore), and **files an issue** on real drift. If you touch the sync chain, keep it **fail-closed and loud** ‚Äî no `exit 0`/`::warning::` silent skips.
- **When in doubt, trust the galleries, not the bundle.** If a consumer reports a missing/wrong icon/component/token, compare against the **live gallery manifests** (android + ios) ‚Äî or the source repos ‚Äî before concluding it doesn't exist. Token parity between the two platforms is checked at merge time and WARNED on drift (release-cadence lag is expected; real drift goes back to the source repos).

## Local dev
`pnpm dev` (run server) ¬∑ `pnpm build` (tsc) ¬∑ `pnpm smoke` (real MCP handshake) ¬∑ `pnpm manifest:fetch` (re-pull the manifest). Releases are **manual** (`release.yml`, `bump: patch|minor|major`) ‚Äî humans own the cadence.
